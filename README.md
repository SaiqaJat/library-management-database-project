## 📌 Overview

This project is a relational Library Management System built using SQL. It simulates real-world library operations such as book issuing, returns, member management, and branch performance tracking.

It demonstrates practical SQL skills including database design, CRUD operations, joins, aggregations, stored procedures, and analytical reporting.

---

## 🏗️ Database Design

The system is built on a relational structure with the following core tables:

- 📚 **Books** – Book details, availability, and pricing  
- 👤 **Members** – Library user information  
- 🧑‍💼 **Employees** – Staff handling book transactions  
- 🏢 **Branch** – Library branch and management data  
- 🔄 **Issued Status** – Records of issued books  
- 📥 **Return Status** – Records of returned books

## ⚙️ Features

- 📖 Manage books (add, update, delete, track availability)
- 👤 Member registration and updates
- 🔄 Issue and return book system
- ⏰ Overdue book tracking
- 🧑‍💼 Employee performance tracking
- 🏢 Branch-wise performance analysis
- 💰 Revenue calculation per category/branch
- ⚙️ Automated workflows using stored procedures

## 📊 SQL Concepts Used

- CRUD operations (Insert, Update, Delete, Select)
- Joins (INNER JOIN, LEFT JOIN)
- GROUP BY & HAVING
- Subqueries
- CTAS (Create Table As Select)
- Stored Procedures (PL/pgSQL)
- Conditional logic (IF/ELSE)
- Date-based analysis
